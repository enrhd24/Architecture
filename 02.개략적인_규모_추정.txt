개략적인 규모 추정 
: 시스템 용량이나 성능 요구사항을 개략적으로 추정해보라는 요구를 받게 된다
: 보편적으로 통용되는 성능 수치상에서 사고 실행을 행하여 추정치를 계산하는 행위
: 어떤 설계가 요구사항에 부합할 거싱니지 보기 위한 것

효솨적인 개략적 규모 추정
: 규모 확장성을 표현하는 데 필요한 기본기에 능숙
: 2의 제곱수나 응닺지연 값, 가용성에 관계된 수치들
// 기본적으로 잘 이해하고 있어야 한다.

1. 2의 제곱수
: 분산 시스템에서 다루는 데이터 양은 커질 수 있으나 "기본"이다. 
: 데이터 볼륨의 단위를 2의 제곱수로 표현하면 어떻게 되는지 알아야 한다.
: 최소 단위는 1바이트, 8비트로 구성된다.
: 아스키 문자 하나가 차지하는 메모리 크기가 1바이트이다.
// 데이터크기를 쓰이는 것을 "데이터 볼륨 단위들"이라고 한다.


2. 모든 프로그래머가 알아야 하는 응답지연 값
: 통상적인 컴퓨터에서 구현된 연산들의 응답지연
: 컴퓨터 연산들의 처리속도를 알기 쉽게 시각화한 개발자가 있음.
// 메모리는 빠르지만 디스크는 아직도 느리다.
// 디스크 탐색은 가낭한 피하라.
// 단순한 압출 알고리즘은 빠르다.
// 데이터를 인터넷으로 전송하기 전에 가능하면 압축하라
// 데이터 센터는 지역에 분산되고, 센터들 간에 데이터 주고받는 데는 시간이 걸린다.


3. 가용성에 관계된 수치들
: 시스템이 오랜 시간 동안 지속적으로 중단 없이 운영될 수 있는 능력
: 100%는 시스쳄이 단 한 번도 중단된 적이 없었음을 의미한다.

SLA 
: 서비스 사업자가 보편적으로 사용하는 용정
: 서비스 사업자와 고객 사이에 맺어진 합의를 의미한다.
: 사업자가 제공하는 서비스의 가용시간이 공식적으로 기술되어 있다.
: 9개의 개수와 시스템 장애시간사이의 관계를 표현한다.

5. 팁
: 가장 중요한 것은 문제를 풀어 나가느 절차이다.
: 면접자가 보고 싶어 하는 것은 여러분의 문제 해결 능력일 것이다.

- 근사치를 활용한 계산 : 면접장에서 복잡한 계산을 하는 것은 어려운 일이다.
- 가정 : "100,000/10"로 간소화 할 수 있다.
- 단위 : 단위를 붙이는 습관을 들이면 모호함을 방지할 수 있다.
- 문제 : QPS, 최대QPS, 저장소 요구량, 캐시 요구량, 서버 수등 추정하는 것
