개략적인 규모 추정
- 성능 수치상에서 사고 실험
- 추정치를 계산하는 행위
- 어떤 설계가 요구사항에 부합할 것인지 보기 위한 것
// 규모 확장성을 표현하는 데 필요한 기본기에 능숙해야 함

01. 2의 제곱 수
- 데이터 볼륨의 단위를 2의 제곱수로 표현하면 어떻게 되는지 알아야 한다.
- 최소 단위는 1바이트, 8바이트
- ASCII 문자 하나가 차지하는 메모리 크기가 1바이트


02. 모든 프로그래머가 알아야 하는 응답지연 값
- 컴퓨터 연산들의 처리 속도가 어느 정보인지 짐작할 수 있도록 해준다.
- 딘 박사는 이 수들을 알기 쉽게 도구를 활용하여 시각화한 수치를 보임.
정리 : 메모리는 빠르지만, 디스크는 아직도 느리다.
     : 디스크 탐색(seek)은 가능한 한 피하라
     : 단순한 압축 알고리즘은 빠르다.
     : 데이터를 인터넷으로 전송하기 전에 가능하면 압축하라
     : 데이터 센터는 여러 지역에 분산되어 센터들 간에 데이터 주고받는 데 시간이 걸림.

03. 가용성에 관계된 수치들
- 가용성 : 시스템이 오랜 시간 동안 지속적으로 중단 없이 운영 될 수 있는 능력
- SLA : 서비스 사업자가 보편적으로 사용하는 용어 [서비스 사업자와 고객 사이에 맺어진 합의]
- 합의 : 사업자가 제공하는 서비스의 가용시간이 공식적으로 기술되어 있따.


04. 팁
- 올바른 절차를 밟느냐가 결과를 내는 것보다 중요하다.
- 여러분의 문제 해결 능력
1. 근사치를 활용한 계산
2. 가정(assumption)들을 적어두자
3. 단위(unit)를 붙이라
4. QPS, 최다QPS, 저장소 요구량, 캐시 요구량, 거버 수 등을 추정하는 것이다.
